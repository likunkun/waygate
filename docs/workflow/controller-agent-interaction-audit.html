主流程

01

Requirements Drafter

生成 Requirements body、summary 和 approval gate；输入 AO ledger、spec、dialogue brief 与 context files。

02

Requirements Gate

人工或 Plannotator 审批；controller preflight 校验 AC layer、AO、4.5、4.9、prototype。

03

Unit Plan Drafter

把 approved Requirements 转成 Test Case Matrix、Document Deliverables Matrix 和 Controller State Patch。

04

Test Strategist

可选独立审查测试策略，生成 strategy、gap report 和 review package；static-only coverage 是 Critical gap。

05

Unit Plan Gate

应用 state patch 并运行 test strategy、AO、Journey、prototype、docs、E2E、env、golden path validators。

06

Builder

读取 approved gates，只执行当前 unit；先补 AC-mapped tests，并完成 required docs deliverables。

07

Refiner

CodeSimplifier 做行为保持型精修；只允许 ok、skipped、changes_requested 或 failed 路由。

08

Reviewer

controller-side review 检查 builder/refiner artifacts、changed files 和失败信号，失败回 Builder。

09

Verifier

执行 Unit Plan commands，写 verification.json、evidence_rows、journey evidence 与 visual evidence refs。

10

Final Acceptance

汇总 Evidence Matrix、Document Status、Prototype Matrix、Scope Audit、Journey Matrix 和 rejection routing。

11

Final Sync / Done

终验通过后同步 task_plan、progress、findings 等状态文档；这不是实现任务。

返工路线

Requirements

已批准需求错误或不完整。失效 Requirements 与 Unit Plan，重新进入 Requirements 人工确认。

Defect Fix

需求正确，但终验发现已完成工作有缺陷。进入 Bug Fix Gate、Bug Fix Agent 和 regression verifier。

Unit Plan

单元范围、测试用例、证据计划或架构选择有缺口。重新生成 Unit Plan。

Implementation

计划正确但实现不完整或错误。当前 unit 标记 incomplete，回到 Builder。

Blocked

环境、数据、权限或证据不足，controller 停在 Final Acceptance 等待人工处理。

关键 Prompt 卡片

Requirements

强制 AC verification layer、AO mapping、4.5 设计/架构、4.8 风险、4.9 基础设施，以及 UI/Web 的 ui-ux-pro-max 与 prototype manifest。

Unit Plan

生成 Test Case Matrix、Document Deliverables Matrix、Controller State Patch；要求真实 E2E entrypoint、golden path 和 prototype conformance 证据计划。

Test Strategist

输出 test-strategy.json、gap report 和 review package；静态检查不能单独覆盖行为验收。

Builder

approved gates 是 source of truth；先读 Test Case Matrix，补测试，再实现当前 unit；Verifier 失败时先复现 exact command。

Refiner / Reviewer

Refiner 只做行为保持型清理；Reviewer 读取 builder/refiner artifacts 和 changed files，不用自然语言总结代替证据。

Verifier / Final

Verifier 产出 evidence rows；Final Acceptance 展示 AO、AC、test case、environment、artifact、scope audit 和 rejection route。

缺口与建议

P1 · 设计/架构进入证据行

给 verification.json evidence rows 增加 product_design_refs 和 technical_architecture_refs，Final Acceptance 同步展示。

P1 · 4.8 风险关闭矩阵

从 Requirements ## 4.8 提取需要关闭的 assumptions/risks，Unit Plan 和 Final Acceptance 逐项显示 closure 状态。

P2 · 4.9 基础设施覆盖矩阵

展示七类基础设施事实如何落到 test setup、verification_env、docs deliverable 或 manual evidence。

P2 · Prompt Registry

把 prompt source 和交互节点登记为正式 workflow 文档，流程变更时同步更新。

P3 · Requirements-stage Test Strategist

按 ROADMAP V0.6.2，把独立 Test Strategist 前移到 Requirements approval 前。
